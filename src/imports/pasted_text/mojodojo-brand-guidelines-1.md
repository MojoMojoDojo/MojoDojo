Create a premium, modern, conversion-focused website and web app for a bakery-dessert brand called “MojoDojo”.

Brand context:
- MojoDojo is a dessert and bakery company with a bold identity, a sleek black background aesthetic, and a signature logo featuring a glowing golden-yellow “M” icon on dark background.
- Use the provided MojoDojo logo as the main brand mark.
- The visual identity should feel premium, memorable, stylish, modern, slightly bold, and hype-worthy — not childish, not generic café, not overdecorated, not overly feminine, and not a bright pastel bakery theme.
- Avoid making the entire interface yellow. Yellow/gold should be used as an accent and brand signal, not as the main fill color everywhere.
- Core palette direction: deep black / charcoal / warm off-black / soft gray / ivory / subtle gold accents inspired by the logo.
- The feeling should be somewhere between luxury dessert brand, minimal street-premium brand, and modern boutique food business.
- The UI should feel clean and expensive, with strong hierarchy, rich photography, subtle motion, clean cards, elegant typography, tasteful rounded corners, premium spacing, and a dark mode-first visual direction.

Main goal:
Design a production-ready website + admin/worker dashboard system that serves two major use cases:
1. Public-facing website for customers to discover the brand, browse products, view gallery, learn the story, contact the company, and place orders.
2. Private internal operations app for owners/admins/workers to manage orders, products, ingredients, recipe usage, revenue, costs, and operational workflows.

Important:
This should not feel like a simple landing page. It should feel like a real digital brand presence with internal tooling behind it. Think complete ecosystem, not brochure site.

Overall design direction:
- Premium dark theme with strong product photography.
- Elegant but punchy typography.
- Strong visual emphasis on the MojoDojo logo and product images.
- Smooth, refined micro-interactions and tasteful motion.
- Use subtle glow, glass, depth, blur, and high-end card styling carefully, not excessively.
- The site should feel sharp, modern, and distinctive.
- The gold/yellow accent from the logo should appear in small highlights, hover states, outlines, badges, icons, key CTAs, section dividers, and subtle gradients.
- Use visual restraint. The brand should feel confident, not loud.
- Bring in a subtle “dojo” energy without becoming cliché martial arts branding. Use the “discipline / craft / mastery / precision” concept rather than literal dojo imagery.
- Introduce one standout brand device that makes the site memorable, such as:
  - a subtle golden line motif derived from the logo shape,
  - a “crafted with discipline” visual rhythm,
  - a premium “batch” / “tray” system in product cards,
  - or a sleek “midnight bakery” vibe with cinematic product presentation.
- The final design should feel unique enough that it does not look like a generic template.

Private app / internal system:
Design a secure private internal app connected to the public website.
This internal system must feel operationally useful, not fake admin fluff.

Roles and permissions:
Create role-based experience with at least:
- Owner
- Admin
- Worker / Staff

Permissions logic:
- Owners have highest access.
- Admins can manage orders, products, recipes, ingredient tracking, staff-facing operational tools.
- Workers can see only what helps them execute work, such as assigned orders, recipe instructions, prep lists, production schedule, and maybe inventory status.
- Workers must NOT see sensitive business metrics like profits, full revenue analytics, cost breakdowns, payment management, or high-level owner settings unless explicitly allowed.
- Workers should not be able to modify sensitive accounting data or final business settings.
- Owners should be able to add/remove roles and manage access.
- Make permission boundaries obvious in the system design.

Admin access / hidden entry:
The admin area should not be prominently exposed in the public navigation.
Design the system assuming the admin area is reached through:
- a direct private URL,
- or a subtle hidden access path,
but do NOT rely on “hidden link = security”.
Real security should come from proper authentication and authorization.
Design secure login patterns, session management, restricted routes, and role-checked access.
Show a clean but serious admin login experience.

This is a major feature.

- If admin manually adds +1 tiramisu order, it should also update ingredient usage.
- Each product should have a recipe profile with ingredients and quantities.
- The system should aggregate ingredient usage across all orders.
- It should show total ingredients needed for a selected date range, day, week, or batch cycle.
- It should help answer questions like:
  - how many eggs are needed this week,
  - how much milk is required for upcoming confirmed orders,
  - what ingredient cost was consumed for a period,
  - what estimated margin/profit exists after ingredient costs.

This section should include:

- session security

- order/payment validation
- rate limiting for forms/logins
- CSRF/XSS/basic abuse prevention considerations
- audit logs for sensitive changes
- secure handling of payment interactions
- no sensitive business data exposed to unauthorized roles
